# Proposal 1200041 milestone 5 close-out

Proposal: [Blaze: Maintenance, Security Improvements, Testing & Assurances for the Blaze SDK](https://milestones.projectcatalyst.io/projects/1200041)

Milestone 5: "Establish Framework for Ongoing Development and Cardano Integration".

## Project close-out

This project strengthened Blaze as a maintained Cardano SDK. The work added provider routing and diagnostics, advanced query caching and event subscriptions, transaction construction safety, typed Aiken scripts, broader unit and e2e coverage, executable examples, and a clearer onboarding path.

The repository now has a documented maintenance framework around that implementation. Packages have explicit ownership boundaries, public work is tracked through GitHub Issues, Cardano integration changes follow a written test and review process, dependency changes are audited before release, and releases run through Changesets and npm trusted publishing.

The project delivered:

- Tested Blockfrost, Maestro, and Kupmios providers with category and operation-level routing.
- Provider debug events that expose selected operations, parameters, results, and failures.
- Cached queries, polling and Ogmios event subscriptions, and query benchmarks.
- Typed datum and redeemer support generated from Aiken blueprints.
- Transaction construction checks, clearer errors, property-based tests, and e2e transaction flows.
- Onboarding, transaction, provider, query, Blueprint, and Aiken contract guides.
- A public roadmap, issue-based task tracking, security policy, and Cardano integration process.
- Automated build, test, coverage, documentation, security, and release workflows.

## Repository and maintenance framework

The monorepo groups public packages under [`packages`](../../../packages) and executable examples under [`examples`](../../../examples). Bun manages the workspace and lockfile, Turbo runs package tasks, Syncpack checks dependency policy, API Extractor tracks public TypeScript surfaces, Docusaurus publishes the guides, and Changesets versions and publishes packages.

[`ROADMAP.md`](../../../ROADMAP.md) lists current and next work and links each concrete item to the public issue tracker. [`CONTRIBUTING.md`](../../../CONTRIBUTING.md#cardano-integration-changes) defines how protocol and provider changes move from an upstream specification or API through package ownership, implementation, tests, documentation, review, and release.

The integration process is exercised by current Conway and SanchoNet support. The emulator uses Cardano SDK `ChainIds.Sanchonet` and the SanchoNet slot configuration rather than local copies. Provider and RPC tests verify the chain ID, network name, timing configuration, and reset behavior.

## Testing and release checks

The maintenance workflow uses the following repository-wide checks:

```sh
bun install --frozen-lockfile
bun syncpack lint
bun run lint
bun run build
bun run typecheck
bun run test
bun --filter @blaze-cardano/e2e test
bun --filter @blaze-cardano/emulator coverage
bun --filter @blaze-cardano/deploy coverage
bun --filter @blaze-cardano/tx test:coverage
bun run docs:prepare
cd docs && bun run build
bun audit --audit-level=high
```

The final local run passed all of these commands. The non-secret e2e suite ran four tests successfully and skipped the credential-gated live transaction test. Coverage was 83.28% for emulator statements, 91.78% for deployment statements, and 72.49% for transaction statements. Function coverage was 89.91%, 97.95%, and 82.38% respectively.

The release workflow builds publishable packages before Changesets publishes them. npm authentication uses trusted publishing rather than a long-lived registry token.

## Security review

The initial dependency scan reported 2 critical, 43 high, 41 moderate, and 3 low advisories. Runtime dependencies with patched releases were updated, and audited root resolutions remove vulnerable transitive versions that remained within older parent ranges.

`bun audit --audit-level=high` now completes without a finding. [`SECURITY.md`](../../../SECURITY.md) provides a private reporting route and explains supported versions, disclosure, lockfile use, dependency review requirements, and release credentials.

A targeted source review also covered the main credential and network boundaries. Live e2e credentials are read from GitHub Secrets, the emulator RPC server binds to `127.0.0.1` by default, and provider debug logging is opt-in with a warning that applications must redact sensitive parameters. No critical or high source finding was identified in those areas.
