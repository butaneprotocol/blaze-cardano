## Acceptance criteria

- The SDK docs explain the main application flow: provider setup, wallet setup, transaction construction, transaction submission, and chain queries.
- Transaction-building docs cover payments, metadata, minting and burning, typed script locking and spending, provider submission, and governance/script redeemer paths.
- Provider docs describe Blockfrost, Maestro, Kupmios, routing, caching boundaries, and the network requests used by each implementation.
- Aiken contract usage is documented through a step-by-step typed script flow. The executable workspace compiles the validator, generates its typed class, deploys a reference script, locks an inline datum, queries the output through `EmulatorProvider`, and spends it with the generated redeemer.
- Contributor guidance explains local setup, focused package checks, API docs generation, Docusaurus builds, PR expectations, and community support.

## Verification

```sh
bun --filter script-deploy-aiken start
bun --filter script-deploy-aiken typecheck
bun --filter advanced-querying start
bun --filter advanced-querying typecheck
bun run docs:prepare
cd docs
bun run build
```

These commands run and typecheck the examples, regenerate the package API docs, and build the Docusaurus site for the SDK, Blueprint, query, transaction, and emulator packages.

## PoA output mapping

### A. Developer documentation and guides

**Output:** Comprehensive developer documentation for transaction building and chain querying with Blaze.

**Acceptance criteria:** A new developer can follow the public onboarding and transaction guides from provider and wallet setup through transaction construction, submission, and chain queries.

**Evidence:** [Developer onboarding](https://blaze.butane.dev/sdk/guides/developer-onboarding/), [transaction building](https://blaze.butane.dev/tx/guides/transaction-building/), [script transactions](https://blaze.butane.dev/tx/guides/script-transactions/), and [advanced querying](https://blaze.butane.dev/query/guides/advanced-querying/).

### B. Base methods, functions, and types

**Output:** Documentation for the base methods, functions, and relevant types used to interact with Blaze.

**Acceptance criteria:** Generated API references are available for the SDK, Blueprint, query, and transaction packages, and the guides link those APIs to practical workflows.

**Evidence:** [SDK API](https://blaze.butane.dev/sdk/api/sdk/), [Blueprint API](https://blaze.butane.dev/blueprint/api/blueprint/), [query API](https://blaze.butane.dev/query/api/query/), and [transaction API](https://blaze.butane.dev/tx/api/tx/).

### C. Provider implementation descriptions

**Output:** Descriptions of each provider implementation, including the network requests each provider makes.

**Acceptance criteria:** The provider documentation explains Blockfrost, Maestro, and Kupmios request paths, pagination and resolution behavior, transaction submission and evaluation, script-reference lookup, routing, and caching boundaries.

**Evidence:** [Provider internals](https://blaze.butane.dev/query/guides/provider-internals/), [provider routing](https://blaze.butane.dev/query/guides/provider-routing/), and [source guide](../../../packages/blaze-query/docs/guides/provider-internals.md).

### D. Interactive Aiken tutorials and examples

**Output:** Interactive tutorials and executable code examples using Aiken contracts.

**Acceptance criteria:** The tutorial is backed by a runnable workspace that compiles an Aiken validator and generates the typed TypeScript interface used by the transaction flow.

**Evidence:** [Aiken contract tutorial](https://blaze.butane.dev/sdk/guides/aiken-contract-tutorial/), [`script-deploy-aiken`](../../../examples/script-deploy-aiken), and [Blueprint guide](https://blaze.butane.dev/blueprint/guides/introduction/).

### E. Step-by-step smart-contract transactions

**Output:** A step-by-step example of building Blaze transactions that interact with an Aiken smart contract.

**Acceptance criteria:** The example deploys a reference script, locks a typed inline datum, queries the output, builds the correct typed redeemer, spends the script output, signs the transaction, and verifies the final emulator state.

**Evidence:** [Aiken contract tutorial](https://blaze.butane.dev/sdk/guides/aiken-contract-tutorial/) and [executable transaction flow](../../../examples/script-deploy-aiken/index.ts).

### F. Provider and query demonstrations

**Output:** Examples that demonstrate the SDK's providers and chain-query APIs.

**Acceptance criteria:** Documentation covers provider selection, operation-level routing, cached reads, UTxO queries, asset queries, datum resolution, script-reference resolution, transaction confirmation, and invalidation after writes. The example runs without network credentials by using deterministic provider fixtures.

**Evidence:** [Advanced querying guide](https://blaze.butane.dev/query/guides/advanced-querying/), [provider routing guide](https://blaze.butane.dev/query/guides/provider-routing/), and [`advanced-querying` example](../../../examples/advanced-querying).

### G. Detailed onboarding material

**Output:** Tutorials written with enough detail for developers who are new to Blaze to build against their own Aiken contracts.

**Acceptance criteria:** The documentation explains prerequisites, installation, provider and wallet setup, Blueprint generation, typed validators, transaction construction, execution, inspection, and common failure points without relying on undocumented project knowledge.

**Evidence:** [Developer onboarding](https://blaze.butane.dev/sdk/guides/developer-onboarding/), [Aiken contract tutorial](https://blaze.butane.dev/sdk/guides/aiken-contract-tutorial/), and [Blueprint generation guide](https://blaze.butane.dev/blueprint/guides/introduction/).

### H. Contribution and community resources

**Output:** Contribution guidelines and community resources.

**Acceptance criteria:** Contributors have public instructions for repository setup, focused tests, typechecking, API documentation generation, documentation builds, pull request expectations, issue reporting, and community support. Public engagement evidence shows continued repository use and contribution.

**Evidence:** [Contribution guide](../../../CONTRIBUTING.md), [issue form](../../../.github/ISSUE_TEMPLATE/blaze_issue.yml), [community engagement](#community-engagement), [M4 review](https://github.com/butaneprotocol/blaze-cardano/pull/355), and [passing documentation build](https://github.com/butaneprotocol/blaze-cardano/actions/runs/29860681269).
