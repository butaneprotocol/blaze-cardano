## Acceptance criteria

- The SDK docs explain the main application flow: provider setup, wallet setup, transaction construction, transaction submission, and chain queries.
- Transaction-building docs cover payments, metadata, minting and burning, typed script locking and spending, provider submission, and governance/script redeemer paths.
- Provider docs describe Blockfrost, Maestro, Kupmios, routing, caching boundaries, and the network requests used by each implementation.
- Aiken contract usage is documented through a step-by-step typed script flow. The executable workspace compiles the validator, generates its typed class, deploys a reference script, locks an inline datum, queries the output through `EmulatorProvider`, and spends it with the generated redeemer.
- Contributor guidance explains local setup, focused package checks, API docs generation, Docusaurus builds, PR expectations, and community support.

## Verification

```sh
bun --filter script-deploy-aiken start
bun --filter script-deploy-aiken typecheck
bun --filter advanced-querying start
bun --filter advanced-querying typecheck
bun run docs:prepare
cd docs
bun run build
```

These commands run and typecheck the examples, regenerate the package API docs, and build the Docusaurus site for the SDK, Blueprint, query, transaction, and emulator packages.
