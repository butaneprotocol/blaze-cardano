# Proposal 1200040 milestone 5 close-out

Proposal: [Blaze: Fully Featured Ledger Emulator, Transaction Builder for the Chang Hard Fork and PlutusV3](https://milestones.projectcatalyst.io/projects/1200040)

Milestone 5: "Production Release and Community Onboarding".

## Project close-out

This project extended Blaze from a transaction-building SDK with a basic local ledger into a deterministic Conway-era emulator and testing environment. The finished implementation covers PlutusV3 execution, DRep and Constitutional Committee flows, governance proposals, voting, ratification, enactment, treasury effects, network presets, configurable ledger timing, and a standalone HTTP API.

The emulator can be used directly in TypeScript tests or run as a separate process. Transactions may be built with Blaze or submitted as signed CBOR from another tool. The same state can be inspected through the provider API or the documented RPC endpoints.

The project delivered:

- Conway transaction and governance support in the transaction builder and emulator.
- PlutusV3 script execution for spending, minting, certificates, withdrawals, voting, and proposal procedures.
- Deterministic DRep, committee, proposal, vote, ratification, enactment, expiry, refund, and treasury behavior.
- Governance inspection APIs and helpers for repeatable application tests.
- A standalone emulator server with OpenAPI documentation and mainnet, preprod, preview, SanchoNet, and custom configurations.
- Benchmarks for wallet registration, payments, script transactions, and governance flows.
- Package guides, generated API references, executable examples, and CI coverage.

## Outcomes

Developers can test governance and PlutusV3 applications without waiting for a public network epoch or running `cardano-node`. Tests can start from known ledger state, advance slots and epochs explicitly, submit transactions, and inspect the resulting governance state. The RPC server makes the same environment available to applications that do not embed the TypeScript package.

The emulator remains deliberately narrower than `cardano-node`. It does not implement consensus, networking, block production, rollbacks, a production mempool, or full stake-pool economics. Keeping those systems out of scope preserves deterministic and fast application tests.

## Testing

The final verification set covers the complete monorepo, the emulator and transaction coverage suites, non-secret e2e flows, documentation generation, and dependency security:

```sh
bun install --frozen-lockfile
bun syncpack lint
bun run lint
bun run build
bun run typecheck
bun run test
bun --filter @blaze-cardano/e2e test
bun --filter @blaze-cardano/emulator coverage
bun --filter @blaze-cardano/tx test:coverage
bun --filter @blaze-cardano/emulator bench
bun run docs:prepare
cd docs && bun run build
bun audit --audit-level=high
```

The final local run passed all of these commands. The non-secret e2e suite ran four tests successfully and skipped the credential-gated live transaction test. Emulator coverage was 83.28% for statements, 89.91% for functions, and 84.39% for lines. Transaction package coverage was 72.49% for statements, 82.38% for functions, and 72.45% for lines.

The credential-backed e2e workflow separately builds, signs, submits, and confirms transactions on Cardano Preview when CI credentials are available.

## Security review

The initial dependency scan reported 2 critical, 43 high, 41 moderate, and 3 low advisories. The critical and high findings were traced through the resolved lockfile. Direct dependencies were updated where patched releases were available, and root resolutions were added for vulnerable transitive packages whose parent ranges still selected old releases.

`bun audit --audit-level=high` now completes without a finding. The [patch Changeset](../../../.changeset/secure-runtime-dependencies.md) prepares the updated runtime dependencies for release. [`SECURITY.md`](../../../SECURITY.md) documents private reporting, supported versions, coordinated disclosure, frozen lockfile use, dependency review requirements, and npm trusted publishing.

A targeted source review also covered the main credential and network boundaries used by this milestone. Live e2e credentials are read from GitHub Secrets, the emulator RPC server binds to `127.0.0.1` by default, and provider debug logging is opt-in with a warning that applications must redact sensitive parameters. No critical or high source finding was identified in those areas.
