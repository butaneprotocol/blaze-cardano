## Standalone server demonstration

The packaged server was started from the repository with `bun --filter @blaze-cardano/emulator serve:rpc -- --host 127.0.0.1 --port 18787`. It reported the RPC URL, OpenAPI document, and Swagger UI after the listener was ready. The following requests were then made against the running HTTP process in sequence.

| Request | Observed result |
| --- | --- |
| `GET /health` | `200`, `{"status":"ok","version":"Emulator"}` |
| `POST /emulator/reset` with the preview preset, a 2-second slot, 4 slots per block, and 120 slots per epoch | `200`, followed by `/emulator/config` returning preview network magic `2` and the requested timing settings |
| `POST /emulator/register` for wallet `alice` with 5 ADA | `200`, and `/emulator/address/alice` returned the same testnet address |
| `POST /emulator/advance` with two blocks | The clock advanced to slot `8`, block `2`, and Unix time `1666656016000` |
| `GET /doc` | OpenAPI `3.1.0` document titled `Blaze Emulator RPC` with 23 paths |

The [RPC server guide](../../../packages/blaze-emulator/docs/guides/rpc-server.md) contains the startup and request examples needed to repeat this run. Raw transaction submission is covered separately by the `accepts pre-built transaction CBOR` integration test in [`packages/blaze-emulator/test/rpc/app.test.ts`](../../../packages/blaze-emulator/test/rpc/app.test.ts). That test assembles and signs a transaction with the lower-level serialization types from `@blaze-cardano/core`, submits the resulting CBOR over the RPC boundary, advances the emulator, and checks the resulting outputs. It does not use Blaze `TxBuilder`.

## Benchmark comparison

The M4 benchmark repeats the command and emulator scenarios from the M1 baseline.

| Scenario | M1 latency avg | M4 latency avg | Change | M1 throughput avg | M4 throughput avg | Change |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| Register mocked wallet | 269,530 ns | 149,966 ns | 44.4% faster | 3,829 ops/s | 6,905 ops/s | 80.3% higher |
| Simple payment | 2,757,496 ns | 2,688,034 ns | 2.5% faster | 418 ops/s | 430 ops/s | 2.9% higher |
| Script lock -> spend | 11,939,709 ns | 11,103,509 ns | 7.0% faster | 92 ops/s | 99 ops/s | 7.6% higher |

The parameter change governance flow, added after M1, averaged 9,848,828 ns and 115 ops/s in the same run.

## Acceptance criteria

- The emulator can run as a standalone HTTP server and can be embedded through `startRpcServer`.
- Clients can reset the emulator with `mainnet`, `preprod`, `preview`, `sanchonet`, or custom network settings.
- Clients can configure slot duration, slots per block, slots per epoch, and protocol parameters, then inspect the active settings through `/emulator/config` and `/emulator/parameters`.
- Transactions can be submitted as raw CBOR through `/emulator/transactions`. The RPC tests cover a signed transaction built with lower-level Cardano serialization primitives instead of Blaze `TxBuilder`, and callers can use the same endpoint with Haskell tooling, another TypeScript builder, or a CLI harness.
- The RPC server exposes OpenAPI metadata and Swagger UI.
- The docs explain server startup, health checks, wallet setup, raw CBOR submission, network configuration, script publishing, governance utilities, and cleanup.
- The M1 registration, payment, and script round-trip benchmark scenarios all improve in average latency and throughput.

## PoA output mapping

### A. REST API server

**Output:** A REST API server that exposes common endpoints for submitting transactions to a remote emulator and querying its state.

**Acceptance criteria:** The standalone server starts successfully, exposes health, configuration, wallet, transaction, state, OpenAPI, and Swagger endpoints, and is exercised through a real HTTP listener and integration tests.

**Evidence:** [Standalone server demonstration](#standalone-server-demonstration), [RPC implementation](../../../packages/blaze-emulator/src/rpc), [RPC guide](../../../packages/blaze-emulator/docs/guides/rpc-server.md), and [RPC tests](../../../packages/blaze-emulator/test/rpc/app.test.ts).

### B. Network configuration

**Output:** Mainnet, testnet, and custom emulator network configurations.

**Acceptance criteria:** The emulator supports mainnet, preprod, preview, SanchoNet, and custom chain IDs, with configurable slot length, slots per block, slots per epoch, and protocol parameters. Tests verify the presets and overrides.

**Evidence:** [`config.ts`](../../../packages/blaze-emulator/src/config.ts), [`ledger-timer.ts`](../../../packages/blaze-emulator/src/ledger-timer.ts), [RPC configuration tests](../../../packages/blaze-emulator/test/rpc/app.test.ts), and [provider tests](../../../packages/blaze-emulator/test/Provider.test.ts).

### C. Documentation and examples

**Output:** Comprehensive endpoint documentation and usage examples.

**Acceptance criteria:** Public documentation covers server startup, health and clock queries, network configuration, wallet setup, raw CBOR submission, script publishing, governance data, and cleanup. The OpenAPI document and Swagger UI are generated by the server.

**Evidence:** [Published RPC guide](https://blaze.butane.dev/emulator/guides/rpc-server/), [source guide](../../../packages/blaze-emulator/docs/guides/rpc-server.md), and [OpenAPI route definitions](../../../packages/blaze-emulator/src/rpc/app.ts).

**Output:** An emulator HTTP boundary that accepts transactions produced by different transaction builders.

**Acceptance criteria:** The transaction endpoint accepts signed transaction CBOR without depending on Blaze `TxBuilder`. The integration test builds and signs a transaction with lower-level Cardano serialization types, submits it through the HTTP API, and verifies the resulting chain state. The benchmark scenarios meet or exceed the M1 baseline.

**Evidence:** [Raw CBOR integration test](../../../packages/blaze-emulator/test/rpc/app.test.ts), [transaction endpoint](../../../packages/blaze-emulator/src/rpc/app.ts), [benchmark comparison](#benchmark-comparison), [feature release](https://github.com/butaneprotocol/blaze-cardano/releases/tag/%40blaze-cardano%2Femulator%400.5.0), and [passing CI](https://github.com/butaneprotocol/blaze-cardano/actions/runs/29860682001).
