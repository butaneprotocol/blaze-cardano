# Catalyst 1200040 milestone 3 evidence

Proposal: [Blaze: Fully Featured Ledger Emulator, Transaction Builder for the Chang Hard Fork and PlutusV3](https://milestones.projectcatalyst.io/projects/1200040)

Current Statement of Milestones: `soms.id=10030`, Milestone 3, "Advanced Governance and PlutusV3 Features Implementation".

## What M3 claims

M3 requires:

- Full Chang governance mechanisms, including DReps and Constitutional Committee flows.
- Script-managed DRep support.
- Comprehensive PlutusV3 support.
- Emulator utilities and documentation for creating deterministic governance test scenarios.
- Transaction-builder support for Chang and PlutusV3 transaction types.
- Integration tests that combine governance and smart-contract interactions.
- Documentation, reproducible examples, benchmark/integration results, and peer review.

## Benchmark snapshot

The benchmark command uses the scenarios in [`packages/blaze-emulator/benchmarks/index.ts`](../../../packages/blaze-emulator/benchmarks/index.ts). These figures are a reproducible local baseline, not a fixed performance guarantee across machines.

| Scenario | Median latency | Median throughput | Samples |
| --- | ---: | ---: | ---: |
| Register mocked wallet | 364,209 ns | 2,746 ops/s | 2,509 |
| Simple payment | 3,540,584 ns | 282 ops/s | 232 |
| Script lock and spend | 13,720,541 ns | 73 ops/s | 63 |
| Parameter-change governance flow | 11,711,667 ns | 85 ops/s | 69 |

In this run, the full parameter-change flow had a median latency of 11.7 ms. That is in the same range as the 13.7 ms script lock-and-spend scenario, so the governance simulation did not add an order-of-magnitude slowdown.
